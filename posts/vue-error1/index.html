问题描述

父组件代码

<template>
  <div class="hello">
    <Counter :num="num"></Counter>
    <p><span>parant:{{num}}</span></p>
  </div>
</template>

<script>
import Counter from './Counter'
export default {
  name: 'HelloWorld',
  data(){
      return{
          num:10
      }
  },
  components:{
      Counter
  },
}
</script>

实现效果

但是当按下加号或减号时，控制台出现warn

vue.runtime.esm.js?2b0e:619 [Vue warn]: Avoid mutating a prop directly since the value will be overwritten whenever the parent component re-renders. Instead, use a data or computed property based on the prop's value. Prop being mutated: "num"

found in

---> <Counter> at src/components/Counter.vue
       <HelloWorld> at src/components/HelloWorld.vue
         <App> at src/App.vue
           <Root>

错误原因

这是因为在vue2.0中移除了组件的props的双向数据绑定功能，如果需要双向数据绑定需要自己来实现。子组件只能被动地接收父组件传递过来的数据，并在子组件内不能修改由父组件传递过来的props数据。

官方文档解释 prop 是单向绑定的：当父组件的属性变化时，将传导给子组件，但是不会反过来。这是为了防止子组件无意修改了父组件的状态——这会让应用的数据流难以理解。

我们可以利用$emit功能实现子组件传递数据给父组件的功能，进而实现双向数据绑定。

解决方法

父组件代码

<template>
  <div class="hello">
    <Counter :num="num" v-on:incre="increment" v-on:decre="decrement"></Counter>
    <p><span>parant:{{num}}</span></p>
  </div>
</template>

<script>
import Counter from './Counter'
export default {
  name: 'HelloWorld',
  data(){
      return{
          num:10
      }
  },
  components:{
      Counter
  },
  methods:{
      increment(){
          this.num++;
      },
      decrement(){
          this.num--;
      }
  }
}
</script>

<style scoped>

</style>

子组件代码

<template>
    <div>
        <button v-on:click="increment()">+</button>
        <button v-on:click="decrement()">-</button>
        <p><span>{{num}}</span></p>
    </div>
</template>

<script>
export default {
    props:{
        num:Number
    },
    data(){
        return{
        }
    },
    methods:{
        increment(){
            this.$emit("incre");
        },
        decrement(){
            this.$emit("decre");
        }
    }
}
</script>
